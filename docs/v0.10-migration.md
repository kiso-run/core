# Migrating from v0.9 to v0.10

v0.10 retires three kiso/core subsystems in one cycle:

- **Wrappers** — the `~/.kiso/wrappers/` directory convention, the
  `kiso wrapper …` CLI, the `wrapper` task type, and the
  `kiso-run/wrapper-*` GitHub org pattern are all gone.
- **Recipes** — the `~/.kiso/recipes/*.md` directory, the
  `kiso recipe …` CLI, and the `recipe_loader` module are gone.
- **`search` task type** — the built-in `searcher` role (bundled
  Perplexity call) is retired. Search is now an MCP server.

If you have a v0.9 config, v0.9 wrappers installed, or scripts that
call `kiso wrapper` / `kiso recipe`, read this document before
upgrading.

There are **no compatibility shims**. `kiso wrapper` / `kiso recipe`
fail with argparse's *invalid choice* error on v0.10. `searcher = "…"`
in `config.toml` fails validation with an upgrade message.

---

## Wrapper feature disposition

Every feature that previously lived inside the wrapper subsystem
ports to a named replacement. Nothing is silently dropped.

| Wrapper feature                              | Where it goes in v0.10                                                                                    |
|----------------------------------------------|-----------------------------------------------------------------------------------------------------------|
| `args_schema` validation                     | MCP method `inputSchema` jsonschema validation                                                            |
| `session_secrets` scoping                    | Per-session MCP client pool with scoped env injection                                                     |
| `env` deploy secrets                         | `~/.kiso/mcp/<name>.env` file integration loaded by the MCP launcher                                      |
| `consumes` file-type routing                 | `x-kiso-consumes` MCP description extension                                                               |
| `usage_guide` narrative                      | MCP tool `description` (native MCP field; `kiso-run/*-mcp` servers populate rich descriptions)            |
| `deps.bin` health at discovery               | `doctor` tool contract for `kiso-run/*` servers; circuit breaker for third-party servers                  |
| `validator.py` hooks                         | Not ported. Semantic repair, when needed, lives inside each MCP server; jsonschema pre-flight covers most |
| Per-user role filtering (`build_planner_wrapper_list`) | `[users.<name>.mcp]` config — allow-list of `server:method` strings                             |
| `needs_install` approval flow                | Extended to MCP + skills (`kiso mcp install --from-url`, `kiso skill install --from-url`)                 |
| `wrapper_recovery` planner module            | `mcp_recovery` planner module                                                                             |

If you depended on a wrapper feature not listed above, it was
scaffolding for the wrapper subsystem itself (discovery, venv
lifecycle, registry fetch) and does not have a v0.10 equivalent —
MCP's native protocol handles its own discovery.

## Wrapper repo → MCP repo mapping

Every `kiso-run/wrapper-*` repo that survived v0.9 is either
renamed to `<capability>-mcp` or deleted. The rewrite is
from-scratch against the official `mcp` Python SDK using the old
wrapper's execution logic as reference only; `run.py` / `kiso.toml`
/ `deps.sh` / `usage_guide.md` are not carried over.

### Renamed (wrapper → mcp)

| Old repo                      | New repo                    | Python package name        | Install invocation (in `mcp.json`)                                                        |
|-------------------------------|-----------------------------|----------------------------|-------------------------------------------------------------------------------------------|
| `kiso-run/wrapper-aider`      | `kiso-run/aider-mcp`        | `kiso-aider-mcp`           | `uvx --from git+https://github.com/kiso-run/aider-mcp@v0.1.0 kiso-aider-mcp`              |
| `kiso-run/wrapper-transcriber`| `kiso-run/transcriber-mcp`  | `kiso-transcriber-mcp`     | `uvx --from git+https://github.com/kiso-run/transcriber-mcp@v0.1.0 kiso-transcriber-mcp`  |
| `kiso-run/wrapper-ocr`        | `kiso-run/ocr-mcp`          | `kiso-ocr-mcp`             | `uvx --from git+https://github.com/kiso-run/ocr-mcp@v0.1.0 kiso-ocr-mcp`                  |
| `kiso-run/wrapper-docreader`  | `kiso-run/docreader-mcp`    | `kiso-docreader-mcp`       | `uvx --from git+https://github.com/kiso-run/docreader-mcp@v0.1.0 kiso-docreader-mcp`      |

### Deleted

| Old repo                      | Replacement                                                                                     |
|-------------------------------|-------------------------------------------------------------------------------------------------|
| `kiso-run/wrapper-browser`    | `@playwright/mcp` (Microsoft-maintained, Tier 1) — now in the default preset                    |

### Newly created

| New repo                      | Replaces                                                                                                   |
|-------------------------------|------------------------------------------------------------------------------------------------------------|
| `kiso-run/search-mcp`         | The retired `search` task type + `searcher` role. Calls Perplexity Sonar directly via OpenRouter.          |

## Recipe migration

The `~/.kiso/recipes/*.md` format is retired, but a one-shot
migration tool reads the legacy format and emits v0.10 skills:

```
kiso-migrate-recipes-to-skills \
    --recipes-dir ~/.kiso/recipes \
    --skills-dir ~/.kiso/skills
```

Each recipe becomes `~/.kiso/skills/<name>/SKILL.md`. After
migrating, delete the `~/.kiso/recipes/` tree. The runtime no
longer reads it.

---

## Config changes

- `searcher = "…"` under `[models]` → **remove the line.** The
  config validator rejects it with a pointer to `kiso-run/search-mcp`.
- Per-user `wrappers = "*"` / `wrappers = ["name", …]` → migrate
  to `mcp = "*"` / `mcp = ["server:method", …]` per M1539.
- `registry_url` in `[settings]` / sysenv → drop it. There is no
  longer a wrapper registry to fetch from.

---

## Directory layout changes

`~/.kiso/` on a freshly-initialised v0.10 install creates the
following subdirectories. `wrappers/` and `recipes/` are **not**
created — existing ones from a v0.9 install are left untouched so
the migration tool can read them, but nothing in v0.10 writes to
them.

```
~/.kiso/
├── connectors/   # installed connector plugins
├── skills/       # installed skills (M1500+)
├── sessions/     # per-session workspaces
├── roles/        # role prompts (user-editable)
├── reference/    # bundled reference docs
├── sys/bin/      # bundled helper binaries
└── sys/ssh/      # managed SSH keys
```

---

## Quick cutover checklist

1. Remove `searcher` from `[models]` in `~/.kiso/config.toml`.
2. Remove `registry_url` from `[settings]` if present.
3. Convert `wrappers = …` under `[users.<name>]` to `mcp = …`.
4. Run `kiso-migrate-recipes-to-skills` if you have recipe files.
5. Replace any `kiso wrapper install <name>` invocations in your
   scripts with the equivalent `uvx` MCP install line from the
   mapping table above, then add the entry to your `mcp.json`.
6. Delete `~/.kiso/wrappers/` and `~/.kiso/recipes/` once you have
   confirmed your workflows run on v0.10.
