# Devplan v0.6 — Plugin Taxonomy Redesign

## Problem Statement

The current naming convention is inconsistent with the broader AI ecosystem. "Skills" in
kiso map to what Claude Code, MCP, and most agent frameworks call "tools" — executable
packages that perform actions. The term "skill" is overloaded and confusing for new users.

Additionally, there's no lightweight concept for prompt-only instructions that guide the
planner without requiring an executable package.

## New Taxonomy

- **plugin** — generic umbrella term for any installable extension
  - **tool** — executable package (browser, aider, search) — what was called "skill"
  - **skill** — lightweight `.md` file with planner instructions (new concept)
  - **connector** — external integration (slack, telegram) — unchanged

## Design Principles

1. **tool** aligns with Claude Code, MCP, and function-calling conventions
2. **skill** becomes a simpler, lower-friction concept — just a markdown file
3. **connector** stays unchanged — already well-defined
4. **plugin** provides the grouping term for CLI (`kiso plugin list`)

## Migration Strategy

- Rename all internal references: `skill` → `tool` for executable packages
- CLI commands: `kiso skill install X` → `kiso tool install X`
- DB tables/columns: `skills` → `tools` (migration script)
- Config keys: `user_skills` → `user_tools`
- Planner/reviewer prompts: update terminology
- Backward compat: accept old CLI commands with deprecation warning for 1 release
- New `kiso skill` CLI for managing `.md` prompt-based skills
