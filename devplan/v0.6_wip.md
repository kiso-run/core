# Devplan v0.6 — Verbose Panel Dedup Fix

## Problem Statement

In verbose mode, IN panels for LLM calls are rendered twice: once immediately by the
inflight rendering system (showing what's being sent while the model is thinking), and
again when the task completes (as part of the full IN+OUT panel pair). The dedup
mechanism (`inflight_in_shown` set in `_PollRenderState`) exists but only works for
exec/skill/search/replan tasks — msg tasks bypass it entirely.

## Problems

### P1 — msg task verbose path bypasses dedup

`_render_msg_task` (cli/__init__.py:669) calls `render_llm_calls_verbose()` directly.
This function in render.py renders ALL IN+OUT panels unconditionally — it has no access
to `state.inflight_in_shown` and cannot skip already-shown IN panels.

All other task types use `_emit_verbose_calls` → `_print_verbose_panels`, which checks
`inflight_in_shown` and skips IN panels already rendered by inflight.

### P2 — msg tasks excluded from intermediate verbose rendering

The intermediate update path (cli/__init__.py:886) has `ttype != "msg"`, which blocks
verbose panel rendering for msg tasks while they're still running. This means when a
msg task runs briefer→messenger sequentially, the briefer OUT panel only appears when
the entire msg task completes (together with all other panels), instead of appearing
as soon as the briefer call finishes.

For exec tasks, OUT panels appear one by one during execution. For msg tasks, they all
dump at the end. This inconsistency makes the output confusing.

---

### M324 — msg task verbose dedup via _emit_verbose_calls

**Problem:** P1

**Files:** `cli/__init__.py` — change `_render_msg_task` to use dedup-aware rendering

**Change:**

1. In `_render_msg_task`, replace the verbose rendering block (lines 668-673):
   ```python
   if verbose:
       verbose_detail = render_llm_calls_verbose(llm_calls_raw, caps)
       if verbose_detail:
           print(verbose_detail)
   ```
   With:
   ```python
   if verbose:
       from cli.render import _parse_llm_calls
       _emit_verbose_calls(task, caps, state, len(_parse_llm_calls(llm_calls_raw)))
   ```
2. This requires adding `state` parameter access — `_render_msg_task` already receives
   it (line 637).

**Test:**
- Existing tests must still pass (no behavior change for non-verbose mode)
- Add test: verify `_render_msg_task` does not call `render_llm_calls_verbose`
  when verbose=True (it should use `_emit_verbose_calls` instead)

### M325 — Allow intermediate verbose rendering for msg tasks

**Problem:** P2

**Files:** `cli/__init__.py` — adjust intermediate update path

**Change:**

1. In the intermediate update block (lines 885-894), separate the review rendering
   (msg tasks don't show review lines) from verbose rendering (msg tasks should show
   verbose panels):
   ```python
   if prev_status == status and prev_status is not None:
       if not quiet:
           if ttype != "msg":
               prev_review = prev_key[1] if prev_key else None
               if review_verdict != prev_review:
                   review_line = render_review(task, caps)
                   if review_line:
                       print(review_line)
           if verbose:
               _emit_verbose_calls(task, caps, state, llm_call_count)
       continue
   ```
2. The key change: move the `verbose` block outside the `ttype != "msg"` guard.
   Review rendering stays msg-excluded; verbose rendering applies to all task types.

**Test:**
- Verify verbose panels are emitted for msg tasks during intermediate updates
- Existing tests must still pass

**Problem:** P1, P2
