# Devplan v0.5 — Role Prompt Token Optimization & Generality

## Problem Statement

Role prompts (`kiso/roles/*.md`) have accumulated verbosity, redundancy, and
language-specific examples (e.g. Italian "installa il connector Y") across v0.2–v0.4.
Every extra token in a system prompt costs latency, money, and context budget on
every single LLM call. Prompt text should be as tight as possible while preserving
all behavioral rules.

Current total: ~33 KB across 11 files. Target: ≤28 KB (≥15% reduction) without
losing any rule or changing behavior.

## Principles

- **No behavior changes** — every rule must be preserved, just expressed more concisely.
- **Language-neutral examples** — no Italian/non-English text in prompts.
- **Remove redundancy** — merge rules that say the same thing in different words.
- **Preserve structure** — keep MODULE markers, section headers, JSON schema docs.
